## 一、基础设施（优先级：P0）

### 1.1 Rust PTY 管理器（后端工程师）
- [ ] **T-001** 实现 `PtyManager` 使用 `portable-pty` 创建 PTY 进程
  - 依赖：无
  - 说明：参考 VSCode PtyHostService，支持 macOS 默认 shell（zsh）
- [ ] **T-002** 实现 PTY 输出异步读取，通过 Tauri event 发送到前端
  - 依赖：T-001
  - 说明：使用 tokio 异步任务读取 master 输出
- [ ] **T-003** 实现 PTY 输入写入和尺寸调整
  - 依赖：T-001
- [ ] **T-004** 实现 PTY 进程退出检测和清理
  - 依赖：T-001

### 1.2 Git 服务（后端工程师）
- [ ] **T-005** 实现 `git status --porcelain=v2 --branch` 输出解析
  - 依赖：无
  - 说明：解析 branch ahead/behind、staged/unstaged/untracked 文件
- [ ] **T-006** 实现 `git diff` 输出获取（unified diff 格式）
  - 依赖：无
- [ ] **T-007** 实现 `git branch` 列表解析
  - 依赖：无
- [ ] **T-008** 实现 `git log` 输出解析
  - 依赖：无

### 1.3 文件监听服务（后端工程师）
- [ ] **T-009** 实现 `FileWatcherManager` 使用 `notify` crate（FSEvent 后端）
  - 依赖：无
  - 说明：监听文件创建/修改/删除/重命名，通过 Tauri event 通知前端
- [ ] **T-010** 实现去抖动（debounce）机制，避免频繁事件
  - 依赖：T-009

### 1.4 项目管理持久化（后端工程师）
- [ ] **T-011** 实现项目列表的 JSON 持久化存储
  - 依赖：无
  - 说明：存储到 Tauri app_data_dir
