## 一、项目管理模块

### US-001 添加项目

**用户故事**：作为开发者，我希望通过多种方式将项目添加到 Star Gazer，以便快速开始管理我的代码仓库。

**验收标准**：
- [ ] AC-1：点击 Sidebar 顶部 "+" 按钮，弹出系统文件选择器，选择文件夹后项目出现在 Sidebar 中
- [ ] AC-2：将文件夹从 Finder 拖拽到 Star Gazer 窗口任意位置，自动识别并添加为新项目
- [ ] AC-3：通过命令面板（Cmd+K）输入 "Add Project"，弹出文件选择器并完成添加
- [ ] AC-4：添加重复项目时给出提示，不重复添加
- [ ] AC-5：添加的项目在应用重启后仍然存在（持久化到 projects.json）

**对应任务**：T-011, T-012, T-014

**补充建议**：TASKS.md 中缺少拖拽添加项目（PRD 5.1.3）和命令面板添加项目的任务，建议在 T-012 中补充这两种添加路径。

### US-002 项目切换与管理

**用户故事**：作为开发者，我希望能方便地在多个项目之间切换、重命名和移除项目，以便灵活管理工作空间。

**验收标准**：
- [ ] AC-1：点击 Sidebar 中的项目条目可以切换当前激活的项目
- [ ] AC-2：右键项目名称弹出上下文菜单，包含：新建 agent（Claude Code/OpenCode/Codex/Custom Command）、文件操作（Open as file card、Open diff）、项目管理（Rename、Reveal in Finder、Close Project、Remove from Star Gazer）
- [ ] AC-3：Close Project 关闭项目但保留在列表中，Remove 从列表中彻底移除
- [ ] AC-4：Remove 操作不删除磁盘上的文件
- [ ] AC-5：项目条目显示项目名称和当前 Git 分支名（小字）
- [ ] AC-6：项目折叠时右侧显示颜色圆点，指示该项目是否有 agent 正在运行

**对应任务**：T-013

**补充建议**：TASKS.md 中 T-013 只提到"项目切换和移除"，未覆盖右键上下文菜单（PRD 5.1.4）、Rename、Reveal in Finder 等功能。建议新增独立任务覆盖右键菜单的完整实现。

## 二、文件树模块

**用户故事**：作为开发者，我希望在 Sidebar 中看到项目的完整文件树，即使面对大型项目也能流畅操作，以便快速定位需要审查的文件。

- [ ] AC-3：每级缩进 16px
- [ ] AC-4：默认隐藏 `.git`、`node_modules`、`.DS_Store` 等常见目录

- [ ] AC-6：10000+ 文件的项目文件树初始化时间 < 500ms

### US-005 文件树 Git 状态融合

**用户故事**：作为开发者，我希望文件树直接展示每个文件的 Git 变更状态，以便无需切换视图就能一瞥了解项目改动情况。

**验收标准**：
- [ ] AC-1：未改动的文件正常显示
- [ ] AC-2：已改动的文件行末右对齐显示 `+X -Y` 行数统计（绿色 +、红色 -）
- [ ] AC-3：新增文件只显示绿色 `+X`
- [ ] AC-4：删除文件的文件名加删除线，显示红色 `-X`
- [ ] AC-5：未跟踪文件（`??`）文件名斜体，前缀小 `?` 符号
- [ ] AC-6：冲突文件前缀红色 `!` 符号
- [ ] AC-7：Git 状态数据实时更新（文件变更后 3 秒内反映）

**对应任务**：T-005, T-016

**补充建议**：TASKS.md 中 T-016 只提到"颜色标记"（modified=黄色等），但 PRD 5.1.6 的要求更丰富：包含 `+X -Y` 行数统计、删除线、斜体、前缀符号等。T-005 也只解析了 status，还需要解析每个文件的 diff stat（行数统计）。建议细化 T-016 或新增任务。

### US-017 PTY 生命周期管理

**对应任务**：T-001, T-004

**补充建议**：T-004 只提到"退出检测和清理"，PRD 5.4.3 还要求保留终端输出、提供重启按钮等。建议细化。
