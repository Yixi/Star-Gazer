设计要点

三栏结构：从左到右依次是固定 sidebar（240px）→ 可关闭的 slide panel（默认 540px）→ 画布（剩余空间）。中间有一条可拖动的 4px 分隔线（画布左边缘那条），鼠标悬停变蓝，左右拖动调整面板宽度。面板完全关闭后，画布会扩展到占用它的空间。

Tab 栏：顶部一条 36px 的 tab 栏，每个 tab 显示文件名、agent 颜色图标、和一个 diff/file 的模式徽章。当前激活的 tab 有顶部蓝色边条和稍亮的背景。Tab 上限无限，超出时右侧出现渐变 fade + ⋯ 符号，横向滚动查看更多。最右边是面板关闭按钮（或按 Esc）。

模式切换：面板工具栏右侧有两个开关——第一个 diff/file 切换内容模式，第二个 split/unified 切换 diff 的左右对比或统一视图。已改动的文件默认进入 diff + split 模式；未改动的文件只能进 file 模式，开关置灰。

面包屑：工具栏左侧显示完整路径 stargazer-app / src / auth.ts，让你知道这是哪个项目的哪个文件——在多项目场景下很重要。

Diff 视图细节：左右分栏都有行号，删除行红底红字、新增行绿底绿字、上下文行正常显示。中间的 @@ -25,6 +28,14 @@ 是 hunk header，蓝色背景分隔不同的改动块，这是 git diff 的标准语法。语法高亮对 TypeScript 关键字、字符串、类型都生效。

画布区域被压缩后的行为：因为面板占了 540px，画布变窄，之前横向并排的三张 agent 卡片现在垂直堆叠。这是刻意设计的——你的主要注意力在面板上的 diff，agent 卡片只需要保持"可见 + 可点"，不需要占据最大视觉权重。如果某个 agent 需要交互（比如 api agent 在等你批准 npm install），你也能看到。

文件树里的激活状态：左边 auth.ts 有一条蓝色竖线 + 淡蓝背景，表示它当前正在面板里被查看。切 tab 时这个激活标记会跟着移动。

关闭面板后的行为

点右上角 × 或按 Esc 关闭面板，画布会平滑扩展到占满 1080px 左右的宽度（sidebar 240 之外的全部）。再次点击任何文件会重新召唤面板并恢复到上次的宽度和 tab 状态。Tab 本身不会因为关闭面板而丢失——关闭的是"视图"，不是"上下文"。
